Huffman Coding & Data Compression

Frequent symbols deserve short codes; rare ones can afford long ones. Huffman's 1952 algorithm builds the provably optimal prefix code by a beautifully simple rule: repeatedly merge the two least-frequent items in the forest. Why merging the two smallest is optimal

Every character starts as its own tree, weighted by how often it appears. Huffman keeps a priority queue of these trees and, at each step, pulls out the two lowest-weight trees and joins them under a new parent whose weight is their sum. That parent goes back into the queue. Repeat until a single tree remains — n − 1 merges for n distinct characters.

The magic is that the two rarest symbols end up deepest in the tree, so they get the longest bit-strings, while common symbols sit near the root with short codes. Because every character is a leaf, no code is a prefix of another — the decoder never needs a separator. Huffman proved this greedy choice yields the minimum possible average code length among all prefix codes; it can only be beaten by giving up the one-integer-bit-per-symbol constraint, which is exactly what arithmetic coding (§20.5) does.

The theoretical floor is Shannon's entropy, H = −Σ p·log₂ p bits per symbol. Huffman always lands within one bit of it, and hits it exactly when every probability is a power of ½ — try the powers of two preset to watch efficiency reach 100%.
